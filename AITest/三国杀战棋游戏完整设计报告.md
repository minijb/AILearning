# 三国杀战棋游戏 — 完整游戏设计分析报告

**分析日期：** 2026-04-01
**项目概述：** 设计一款融合"逆塔防放置 + 战棋自动战斗 + 三国杀技能系统"的三国题材战棋游戏
**参考游戏：** 循环勇者 (Loop Hero)、王国保卫战 (Kingdom Rush)、背包乱斗 (Backpack Battles)

---

## 执行摘要

本报告基于对三款参考游戏（循环勇者、王国保卫战、背包乱斗）核心机制的系统性研究，结合三国杀IP的技能体系，设计一款创新的三国题材战棋游戏。核心创新在于：**将逆塔防的"放置-自动战斗"循环、战棋的走格战斗机制、以及三国杀的武将技能系统三者有机融合**，创造一种"放置战棋"的新品类。

游戏核心循环为：**布置武将阵位 → 武将自动走格战斗 + 手动释放技能 → 击败敌人获取资源 → 升级武将与技能 → 挑战更高难度关卡**。通过这一循环，实现了低操作门槛与高策略深度的平衡。

## 一、参考游戏深度研究

### 1.1 循环勇者 (Loop Hero) — 逆塔防机制解析

**核心定位：** 反向塔防 + 永久局外 roguelike

循环勇者颠覆了传统塔防"建造防御塔攻击敌人"的逻辑，转而采用"玩家放置地形召唤敌人，英雄在环形路径上自动行走并击杀敌人"的逆塔防模式。

#### 核心机制拆解

**地图与循环系统：**
- 英雄在固定的环形路径上自动循环行走，无需玩家操作移动
- 地图由玩家放置的"地块"(Tile) 组成，不同地块产生不同效果
- 地块分为三类：资源类（草地、森林采集点）、敌人召唤类（战场、墓地召唤怪物）、增益类（河流、沼泽减速敌人）
- 玩家通过战略性放置地块来控制敌人类型、密度和战斗节奏

**战斗系统：**
- 战斗完全自动进行，英雄根据装备自动攻击路径上的敌人
- 敌人类型与放置的地块强关联——放置墓地地块召唤死灵，战场地块召唤战士
- 引入了"敌人密度"概念：过多敌人召唤地块会导致敌人过强，过少则收益不足
- 关键设计：玩家不直接控制战斗，而是通过调整地图布局来间接影响战斗结果

**局外成长（营地系统）：**
- 营地(Camp) 是局外永久建造界面，提供不随单局重置的增益
- 可建造设施：磨坊（增加资源掉率）、篝火（回复生命）、铁匠铺（装备耐久）、瞭望塔（显示敌人信息）
- 营地升级消耗单局获得的资源，形成局内-局外循环
- 循环勇者巧妙地让玩家在"战斗冒险"（低压）和"规划布局"（高压）之间交替

**资源与循环：**
```
击杀敌人 → 获得物资（布料、皮革、矿材、宝石）
    ↓
物资用于建造/升级营地设施
    ↓
营地升级提供全局加成（掉率、回复、装备耐久）
    ↓
更强属性挑战更难的 Boss 循环
```

**设计精髓提炼：**
1. **策略与执行分离**：放置地块是策略层（可暂停规划），自动战斗是执行层（低操作压力）
2. **间接控制哲学**：不直接控制角色，而是通过环境布局影响结果
3. **资源闭环**：所有资源都有明确的用途和去处，无沉没成本
4. **适度随机**：地图循环但敌人装备有随机性，提供重复可玩性

### 1.2 王国保卫战 (Kingdom Rush) — 经典塔防机制解析

**核心定位：** 经典塔防 + 英雄操作 + 深度升级

王国保卫战是塔防品类的标杆作品，其设计精髓在于将"建造-防御-升级"的循环做到了极致的策略深度和情感满足。

#### 核心机制拆解

**防御塔系统（四基础塔 × 多升级分支）：**
- **箭塔**：对空优先，攻速快，伤害一般 → 升级路线：优先对空 / 强化对地 / 增强伤害
- **魔法塔**：AOE伤害，攻击较慢，对付成群敌人 → 冰冻 / 闪电 / 火焰三系升级
- **炮塔**：高伤害，AOE，对付重甲敌人 → 榴弹 / 火箭 / 集束弹升级
- **兵营**：阻挡敌人前进，士兵可手动释放技能 → 重装 / 游侠 / 萨满等分支
- 每个塔有3-4个升级阶段，总计超过16种塔类型

**英雄系统：**
- 每个关卡可携带一名英雄，英雄是可控单位
- 英雄技能（通常3个）需要手动释放：眩晕、治疗、范围伤害等
- 英雄可以移动到地图任意位置，提供战术支援
- 英雄本身也有升级路线，提供成长线
- 设计意图：让玩家在"塔防自动防御"的基础上，有手动操作的空间和爽感

**关卡与波次设计：**
- 关卡内分为多个波次(Wave)，每个波次有特定的敌人类型和数量
- 波次之间有短暂间隙，供玩家调整布局或购买新塔
- Boss 敌人需要特殊战术（如需要特定塔类型或英雄技能）
- 地图地形多样化：有多条敌人路径、斜坡加速、空中敌人等

**经济系统：**
- 金币来源：击杀敌人、关卡完成奖励、成就奖励
- 金币用途：建造塔、升级塔、购买英雄技能
- 战略决策：是在前面省钱速出关键塔，还是存钱憋高阶塔？

**设计精髓提炼：**
1. **经典循环的可预测满足**：建造 → 防御 → 升级 → 更大挑战 → 更好奖励
2. **英雄操作层的添加**：在自动防御塔之上增加手动操作层，增加参与感
3. **升级树的多样性**：每个塔的升级分支都提供不同的策略选择
4. **波次节奏设计**：通过敌人波次的时间安排创造紧张感和策略调整窗口

### 1.3 背包乱斗 (Backpack Battles) — 背包管理机制解析

**核心定位：** 背包摆放 + 自动对战 + 经济管理

背包乱斗将"背包管理"这一看似简单的机制（类RPG道具栏）提炼为核心玩法，创造了一种"摆放下棋"的新体验。

#### 核心机制拆解

**背包系统（核心创新）：**
- 玩家在一个有限尺寸的背包网格中摆放物品
- 每个物品占用不同大小的格子（1×1、2×1、1×2、2×2 等）
- 物品摆放位置影响属性：某些物品放在特定位置获得"星位加成"
- 物品可以"合成"：将相邻的特定物品组合成更高级道具
- 背包空间有限，摆放需要策略性取舍

**道具系统：**
- 武器、护甲、饰品、消耗品等分类
- 同类道具可叠加/升级（如多把剑合成为更强的剑）
- 道具之间存在"联动"：相邻放置产生额外效果
- 道具稀有度系统：普通 → 精良 → 稀有 → 史诗 → 传说

**自动战斗系统：**
- 玩家完成背包摆放后，进入自动战斗阶段
- 战斗完全自动进行，胜负取决于背包配置
- 战斗可视化：可以看到角色的移动、攻击、技能释放
- 提供重播功能，让玩家分析失败原因

**经济系统（Roguelike 循环）：**
```
开局选择职业/初始装备
    ↓
商店阶段：购买、出售、重新排列背包
    ↓
战斗阶段：与其他玩家/AI 的配置自动对战
    ↓
结算：根据排名获得金币，购买更好的装备
    ↓
循环往复，直到死亡或达成目标
```

**设计精髓提炼：**
1. **操作即策略**：摆放背包这个"操作行为"本身就是策略决策
2. **所见即所得**：背包配置直接可视化，配置过程就是思考过程
3. **经济博弈**：何时购买、何时出售、如何配置，决策丰富
4. **BD 构建空间**：道具联动 + 星位加成创造极大的构建自由度

## 二、三款参考游戏的机制对比与融合点分析

### 2.1 机制矩阵对比

| 设计维度 | 循环勇者 | 王国保卫战 | 背包乱斗 | 三国杀战棋（目标） |
|---------|---------|-----------|---------|-----------------|
| **放置机制** | 放置地块召唤敌人 | 建造塔攻击敌人 | 摆放物品配置属性 | 放置武将占据格子 |
| **战斗模式** | 自动战斗（英雄循环走格） | 半自动（塔自动+英雄手动） | 完全自动（基于配置） | 自动走格 + 手动技能 |
| **策略层** | 地图布局规划 | 塔位选择与升级路线 | 物品选择与摆放 | 阵位布置 + 技能释放时机 |
| **执行层** | 自动执行（无需操作） | 塔自动攻击 + 英雄操作 | 自动战斗（无操作） | 武将被动自动，技能手动 |
| **成长系统** | 营地（局外）+ 装备（局内） | 关卡解锁 + 塔升级 | 每局经济循环 | 武将升级 + 技能升级 + 装备 |
| **随机性来源** | 敌人掉落、地块效果 | 关卡设计、敌人波次 | 商店随机、战斗匹配 | 关卡敌人、技能随机 |
| **玩家压力** | 低（可暂停规划） | 中（波次实时管理） | 中（经济决策压力） | 中低（可暂停，技能手动） |
| **循环周期** | 长（局外永久养成） | 中（关卡制） | 短（每局 15-30 分钟） | 中长（关卡 + 局外养成） |

### 2.2 三大机制的融合路径

**1. 循环勇者的"间接控制"应用到战棋：**
- 不让玩家直接操控每个单位的每一步行动
- 玩家决策层面：武将站位、技能配置、阵型编排
- 执行层面：武将自动在战棋格子上移动和攻击
- 玩家通过"阵位设计"来间接控制战局

**2. 王国保卫战的"英雄操作层"叠加到放置系统：**
- 在自动战斗的基础上，增加手动释放技能的层次
- 技能释放需要有策略性——何时放、给谁放、放哪个技能
- 避免技能变成"无脑点击"，需要有冷却管理和优先级判断

**3. 背包乱斗的"配置即策略"应用到武将系统：**
- 武将的技能配置如同背包中的物品摆放
- 武将之间存在联动（类似道具相邻加成）
- 武将组合（阵型）本身就是一种策略构建
- 提供武将之间的"合击技能"或"羁绊系统"

### 2.3 融合后的目标游戏类型定位

**"放置战棋"（Placement Auto-Chess）：**
- 核心区别于传统战棋：不需要逐格操控单位移动
- 核心区别于传统放置：不是纯挂机，有战棋格子和技能操作
- 核心区别于塔防：不是建造防御工事，是放置作战单位
- 类比：循环勇者（放置机制）× 战棋（格子系统）× 三国杀（技能系统）

## 三、三国杀IP核心要素分析

### 3.1 三国杀技能系统研究

三国杀作为国内最成功的卡牌桌游之一，其技能系统设计极为精妙。将这些技能移植到战棋环境中，需要进行系统性的分类和适配。

**技能分类体系：**

**A. 主动技能（需要玩家手动触发）**

| 技能类型 | 代表技能 | 战棋化适配 | 设计考量 |
|---------|---------|-----------|---------|
| **伤害类** | 杀、南蛮入侵、万箭齐发 | 指向性单体/范围伤害 | 需要目标选择界面 |
| **控制类** | 乐不思蜀、兵粮寸断 | 减速/禁止移动/跳过回合 | 影响AI路径规划 |
| **辅助类** | 桃、桃园结义 | 回复生命、全队增益 | 需要血量显示支持 |
| **过牌类** | 过河拆桥、顺手牵羊 | 移除对方buff、偷取资源 | 增加交互深度 |
| **转化类** | 武圣（红桃当杀）、连环 | 改变攻击属性、传递伤害 | 需要状态系统支持 |

**B. 被动技能（自动触发）**

| 触发方式 | 代表技能 | 战棋化适配 |
|---------|---------|-----------|
| **受到伤害时** | 铁骑（判定）、反馈 | 反击机制 |
| **造成伤害时** | 烈弓（根据体力差加成）、裸衣 | 增伤机制 |
| **回合开始/结束** | 观星、据守 | 回合触发型效果 |
| **击杀时** | 标包关羽（ overkill 反馈） | 击杀奖励 |
| **濒死时** | 急救、不屈 | 生存机制 |

**C. 觉醒技能（条件触发）**

| 触发条件 | 代表技能 | 战棋化适配 |
|---------|---------|-----------|
| **体力值降至某阈值** | 咆哮（进入咆哮状态）、离线托管 | 状态转换 |
| **累计击杀数** | 疐徒（杀敌后摸牌） | 击杀计数器 |
| **特定行动次数** | 铁骑（累计使用杀次数） | 行动计数器 |

### 3.2 三国杀武将体系分析

**武将品质分级（可移植到战棋）：**

| 分级 | 特点 | 战棋武将设计方向 |
|-----|------|----------------|
| **史诗级 (SP)** | 3-4个复杂技能，多重机制组合 | 核心武将，限1-2个，需要高操作水平 |
| **精品级 (SSR)** | 2-3个技能，有明确的定位 | 主力武将，各有特色 |
| **普通级 (R)** | 1-2个简单技能，定位明确 | 填充阵容，提供基础能力 |
| **N卡级** | 1个被动或极简主动 | 背景板，可用于合成或升级材料 |

**武将职业分类（战棋化）：**

| 职业 | 战斗风格 | 代表武将 | 核心特性 |
|-----|---------|---------|---------|
| **先锋** | 前排肉盾，高生命 | 曹操、司马懿 | 嘲讽、减伤、反伤 |
| **战士** | 近战输出，平衡属性 | 关羽、张飞、赵云 | 高攻、多杀、连击 |
| **谋士** | 远程/辅助，控场为主 | 诸葛亮、郭嘉、荀彧 | 增益、减益、控制 |
| **弓手** | 远程物理，高伤害 | 黄忠、太史慈 | 狙击、穿透、标记 |
| **刺客** | 突进收割，高爆发 | 吕布、夏侯惇 | 突进、斩杀、真伤 |
| **医师** | 治疗支援，团队增益 | 华佗、左慈 | 治疗、复活、驱散 |

### 3.3 三国杀身份系统与阵营系统的适配

**方案一：身份系统适配**
- 主公：全局指挥，额外的光环技能
- 忠臣：主公存活时获得加成
- 反贼：击杀主公获胜，击杀奖励更高
- 内奸：存活到最后的隐藏目标（单人模式可移除）

**方案二：阵营系统适配（更推荐战棋场景）**
- 魏国：防御型，高存活，技能多与生命值相关
- 蜀国：进攻型，高爆发，技能多与攻击次数相关
- 吴国：控制型，多过牌和判定，技能多与手牌相关
- 群雄：平衡型，技能多样化，适应性强

**阵营羁绊系统：**
- 同阵营武将相邻时触发"羁绊"加成
- 例：桃园结义（刘备+关羽+张飞同时在场，全队+10%属性）
- 例：五虎上将（5名蜀国武将同时在场，技能冷却-20%）

### 4.2 系统层次结构

```
L1 核心循环层（游戏必须有的最小系统集）
│
```

## 五、核心战斗系统设计

### 5.1 战棋格子系统

**地图规格：**
- 标准战斗地图：8列 × 6行（宽×高），共48格
- 敌方起点：地图右侧边缘
- 我方终点：地图左侧边缘
- 武将放置区：左侧2列（共16格，可放置4-8名武将）
- 敌人路径：主要从右侧向左侧推进，部分关卡有分支路径

**格子属性：**
| 格子类型 | 效果 | 视觉表现 |
|---------|------|---------|
| 普通格 | 无特殊效果 | 地面 |
| 草地格 | 移动消耗+1 | 绿色草地 |
| 河流格 | 不可通行 | 蓝色水流 |
| 陷阱格 | 经过时受到伤害 | 地面裂缝 |
| 增益格 | 停留时获得属性提升 | 发光地面 |
| 遮挡格 | 远程攻击无法直接命中 | 石头/树木 |

**移动规则：**
- 武将每回合移动1-3格（取决于兵种类型）
- 移动路径采用曼哈顿距离寻路
- 无法穿越敌方单位（阻挡机制）
- 武将之间不可重叠放置

### 5.2 自动战斗流程

**回合制 + 实时元素的混合设计：**

```
┌──────────────────────────────────────────────────────────┐
│                    战斗时间轴                              │
│                                                          │
│  [武将回合] ──→ [武将回合] ──→ [武将回合] ──→ [敌人回合] │
│      ↑                                               │   │
│      └──────────────────── 循环 ────────────────────┘   │
│                                                          │
│  每个"回合"约 3-5 秒（可加速/减速）                       │
└──────────────────────────────────────────────────────────┘
```

**自动战斗行为优先级（每个武将的AI）：**

```
优先级 1: 优先攻击范围内的敌方单位
优先级 2: 若无攻击目标，向最近的敌方单位移动
优先级 3: 若在攻击范围内，释放自动触发的被动技能
优先级 4: 回合结束，等待玩家指令或进入下一回合
```

**手动操作窗口：**
- 战斗开始前的"布置阶段"（15-30秒）：放置/调整武将位置
- 战斗中的"技能释放"：玩家在武将回合可释放主动技能
- 战斗中的"暂停"：随时可暂停进行策略调整
- 战斗中的"加速/减速"：1倍/2倍/3倍速度切换

### 5.3 伤害计算公式

**基础伤害公式：**
```
最终伤害 = 攻击方攻击力 × 技能系数 × 克制系数 × (1 + 暴击率 × 暴击伤害加成)
```

**属性克制：**
| 攻击方兵种 | 被克制 | 克制 | 无克制 |
|----------|-------|------|-------|
| 步兵 | 弓兵 | 骑兵 | 谋士 |
| 骑兵 | 步兵 | 弓兵 | 谋士 |
| 弓兵 | 谋士 | 步兵 | 骑兵 |
| 谋士 | 骑兵 | 步兵 | 弓兵 |

**伤害浮动：±10%随机范围**

### 5.4 怒气/能量系统

**怒气获取方式：**
| 方式 | 怒气获取 | 说明 |
|-----|---------|------|
| 普通攻击 | +10点 | 每次普攻获得 |
| 受到伤害 | +5点 | 每次受伤获得 |
| 击杀敌人 | +20点 | 击杀时获得 |
| 回合结束 | +5点 | 被动积累 |

**怒气消耗（主动技能）：**
| 技能等级 | 怒气消耗 | 设计意图 |
|---------|---------|---------|
| 普通技能 | 20-30点 | 频繁使用，3-4回合一次 |
| 高级技能 | 40-60点 | 核心输出，需要规划 |
| 大招 | 80-100点 | 扭转战局，需要积累 |

**怒气上限：100点（初始）→ 可通过装备/天赋提升至150点**

### 6.2 技能配置系统（类背包乱斗）

每个武将最多可携带 **3个技能**：
- 1个被动技能（自动触发）
- 1个主动技能（手动释放）
- 1个终极技能（大招，需怒气满释放）

**技能槽位：**
```
┌─────────────────────────┐

│  ├─────┤                │

│     [升级] [替换] [遗忘] │

```

### 6.3 技能升级系统

**升级材料：** 同名技能卡 或 通用技能经验
**升级效果：** 每级 +15% 效果数值，-5% 冷却时间（最低50%）

**技能突破（觉醒）：**
- 技能达到5级后，可消耗对应武将碎片进行"突破"
- 突破效果：解锁技能的第二段效果或增强原效果
- 例：咆哮（每回合2次普攻）→ 觉醒：每回合3次普攻 + 每次普攻10%概率再次普攻
