## 执行摘要

**核心定位：** 本游戏定位为"放置战棋"(Placement Auto-Chess) 新品类，通过融合循环勇者的逆塔防放置机制、王国保卫战的防御单位概念、背包乱斗的配置即策略理念，以及三国杀的武将技能系统，创造一种"放置武将阵位 → 自动走格战斗 → 手动释放技能"的新型游戏体验。

**三大核心创新：**
1. **品类创新**：填补"有策略深度的放置游戏"市场空白，不与传统放置RPG或手操战棋直接竞争
2. **玩法融合**：将战棋格子、自动战斗和三国杀技能有机结合，策略深度与操作便捷性平衡
3. **IP优势**：三国杀品牌拥有深厚的用户基础和成熟的技能系统，可快速建立差异化

**核心循环：** 布置武将阵位 → 启动战斗 → 武将自动走格战斗 + 玩家手动释放技能 → 击杀敌人获得资源 → 武将升级/装备强化/技能升级 → 挑战更高难度关卡

## 一、类型架构分析

### 1.1 新品类定义

```
放置战棋 = 循环勇者（放置机制）× 战棋（格子系统）× 背包乱斗（配置即策略）× 三国杀（技能系统）
```

**与现有品类的核心区别：**

| 品类 | 操控方式 | 策略焦点 | 战斗时长 |
|-----|---------|---------|---------|
| 传统战棋 | 逐格操控 | 走位、攻击时机 | 15-30分钟 |
| 塔防 | 建造防御塔 | 塔位选择、升级顺序 | 5-15分钟 |
| 自走棋 | 配置阵容 | 阵容搭配、站位 | 20-40分钟 |
| 放置RPG | 纯挂机 | 数值积累 | 随时挂机 |
| **放置战棋** | **放置+手动技能** | **阵位+技能+羁绊** | **2-5分钟** |

### 1.2 四层系统架构

```
L1 核心循环层：战斗引擎 + 放置系统 + 技能系统 + 关卡系统
L2 成长循环层：武将养成 + 装备系统 + 阵法系统 + 羁绊系统
L3 策略深度层：天赋系统 + 觉醒系统 + 难度系统 + 无尽模式
L4 社交商业层：联盟系统 + 竞技系统 + 赛季系统 + 付费系统
```

### 1.3 核心机制清单

| 类别 | 核心机制 | 设计来源 |
|-----|---------|---------|
| 战斗机制 | 战棋格子、自动走格、怒气系统、兵种克制、技能触发 | 战棋 + 三国杀 |
| 放置机制 | 武将放置、阵型阵法、羁绊激活、地形增益 | 王国保卫战 + 背包乱斗 |
| 成长机制 | 武将升级、突破/觉醒、装备强化、天赋系统 | 通用设计 |
| 经济机制 | 多货币、资源闭环、抽卡保底、付费深度 | F2P最佳实践 |

## 二、吸引力因素分析

### 2.1 核心循环图谱

```
阵位布局（策略）
       ↓
启动战斗（期待）
       ↓
观看武将自动战斗 + 释放手动技能（观看+决策）
       ↓
击杀敌人/通关结算（奖励反馈）
       ↓
获得经验/金币/掉落（成长反馈）
       ↓
武将升级/装备强化（更强→挑战更难）
```

### 2.2 心理机制应用

| 机制 | 应用设计 |
|-----|---------|
| 心流理论 | 策略层（布局）与执行层（自动战斗）分离，保持适度挑战 |
| 预期理论 | 即时奖励（伤害数字）+ 中期奖励（通关）+ 长期奖励（突破） |
| 损失厌恶 | 限时优惠、每日刷新、FOMO机制 |
| 社会认同 | 排行榜、分享奖励、联盟贡献榜 |

### 2.3 留存设计

| 阶段 | 设计重点 | 关键机制 |
|-----|---------|---------|
| D1留存 | 快速建立成就感 | 新手十连保底SSR、首战告捷 |
| D7留存 | 建立长期目标 | 军衔系统、关卡解锁 |
| D30+留存 | 社区和深度 | 赛季制、新武将、高难度挑战 |

## 三、竞品分析

### 3.1 市场定位

处于放置游戏、战棋游戏、塔防游戏、自走棋四类游戏的交叉区域，以"放置战棋"新品类切入市场。

### 3.2 差异化矩阵

| 维度 | 本游戏 | 放置RPG | 手操战棋 | 自走棋 |
|-----|-------|---------|---------|-------|
| 操控方式 | 放置+手动技能 | 纯挂机 | 全手操 | 纯配置 |
| 战斗表现 | 战棋格子自动战斗 | 角色站桩 | 角色移动 | 棋子移动 |
| 技能释放 | 手动释放主动技能 | 无手动 | 手动全部 | 无技能 |
| IP特色 | 三国杀技能系统 | 无IP | 无IP | 无IP |

### 3.3 SWOT分析

| 优势 | 劣势 | 机会 | 威胁 |
|-----|-----|-----|-----|
| 三国杀IP护城河 | 市场认知成本高 | 放置战棋无强力竞品 | 大厂可能跟进 |
| 品类创新先发 | 开发成本高 | 三国杀用户可转化 | IP授权不确定性 |
| 策略深度高 | 平衡难度大 | 移动端轻度需求增长 | 放置类同质化 |

## 五、经济系统设计

### 5.1 货币体系

| 货币 | 获取方式 | 主要用途 | 可付费 |
|-----|---------|---------|-------|
| 金币 | 关卡、任务、出售 | 武将升级、装备强化 | 可（有周上限） |
| 元宝 | 充值、月卡、成就 | 招募武将、商城购买 | 主要付费货币 |
| 战功 | PVP竞技场 | 竞技场商店 | 不可 |
| 军功 | 战役通关 | 军衔升级 | 不可 |

### 5.2 抽卡系统

| 品质 | 单抽概率 | 累计保底 | 定向保底 |
|-----|---------|---------|---------|
| SP | 0.5% | 第180抽必出 | 第360抽可选 |
| SSR | 5% | 第90抽必出 | 第180抽可选 |
| SR | 25% | 无 | 无 |
| R | 69.5% | 无 | 无 |

### 5.3 付费用户分层

| 用户类型 | 月均消费 | 占比 | 特点 |
|---------|---------|-----|------|
| 零氪 | 0 | 70% | 完整游戏体验，纯肝 |
| 月卡党 | 30元 | 20% | 月卡+战令 |
| 小/中氪 | 100-1500元 | 8% | 战令+抽卡+体力 |
| 重氪+鲸鱼 | 3000元+ | 2% | 全收集+皮肤 |

### 5.4 经济设计原则

1. **不Pay-to-Win**：PVP使用统一数值BUFF，不使用养成属性
2. **透明保底**：所有概率公开，保底次数明确显示
3. **付费即服务**：付费获得更好体验，而非强制需求
4. **价值感知**：首充超低价建立锚点

## 六、UX体验设计

### 6.1 新手引导流程

| 时间点 | 操作 | 反馈 | 情感设计 |
|-------|------|------|---------|
| 0:00 | 启动游戏 | Logo动画 | 品牌感知 |
| 0:30 | 选择阵营 | 魏/蜀/吴/群选择 | 故事引导 |
| 1:00 | 教程战斗 | 自动引导放置 | "排兵布阵" |
| 2:00 | 首次普攻 | 伤害数字飞出 | 战斗体验 |
| 3:00 | 首次技能 | 技能特效爆发 | 技能爽感 |
| 4:00 | 通关结算 | 3星展示+奖励 | 成就感 |
| 5:00 | 首次升级 | 属性数字滚动 | 成长反馈 |

### 6.2 信息架构

```
一级导航（底部Tab）：战役 | 无尽 | 竞技 | 武将 | 更多

战役Tab：主线关卡 + 精英关卡 + 每日挑战
无尽Tab：无限波次挑战 + 排名
竞技Tab：排位赛 + 巅峰赛
武将Tab：武将列表 + 养成 + 阵容配置
更多Tab：联盟 + 成就 + 军衔 + 商城 + 战令 + 天赋 + 设置
```

### 6.3 情感化设计要点

**战斗反馈：**
- 普通攻击：白色数字，无特效
- 暴击：金色数字，屏幕微震
- 技能伤害：蓝色数字，技能特效
- 击杀：紫色大数字，击杀动画

**成长反馈：**
- 升级：角色发光 + 属性滚动 + "+1"飞出
- 突破：全屏变暗 + 星级点亮 + 光芒特效
- 羁绊激活：图标高亮 + 效果展示

## 七、核心设计建议

### 7.1 最重要5条设计建议

**建议1：坚持"策略与执行分离"的核心设计理念**
- 依据：循环勇者的成功证明"间接控制"能创造独特的策略乐趣；本游戏的核心差异在于放置阵位（策略层）与自动战斗（执行层）的分离
- 执行：确保武将移动、基础攻击完全自动，技能释放是唯一的手动操作点
- 风险：过多手动操作会破坏放置游戏的轻松感

**建议2：优先打磨核心战斗循环的平衡性**
- 依据：作为战棋+放置的混合品类，战斗数值平衡是生死线；王国保卫战经过多代才完善平衡
- 执行：上线前进行至少3轮封闭测试，每次测试收集战斗数据并调整数值
- 风险：数值膨胀会导致游戏后期失去挑战性，数值过低会让玩家感觉"刮痧"

**建议3：三国杀技能的战棋化要"简化但不失精髓"**
- 依据：三国杀技能复杂（判定、距离、手牌等），直接移植到战棋会很繁琐
- 执行：保留技能的核心效果（伤害/控制/治疗），简化触发条件（去掉手牌判定）
- 风险：过度简化会让三国杀粉丝失望，过于复杂会吓退新玩家

**建议4：建立赛季重置机制防止数值膨胀**
- 依据：放置类游戏常见问题是数值膨胀导致后期内容失去意义；循环勇者通过营地重置解决
- 执行：每2个月赛季重置时，提供"赛季商店"让玩家兑换永久道具，同时重置PVP数值
- 风险：赛季重置可能让休闲玩家感到"白肝了"

**建议5：PVP必须使用统一BUFF保证公平性**
- 依据：F2P游戏最大的玩家流失原因是"付费碾压"导致的挫败感
- 执行：竞技场使用"预设属性"模式，玩家选择武将后获得统一加成属性进行对战
- 风险：预设模式可能让养成收集失去意义，需要通过皮肤/称号等提供炫耀出口

### 7.2 差异化核心优势

| 优势 | 描述 | 护城河强度 |
|-----|------|----------|
| IP优势 | 三国杀品牌知名度高，技能系统成熟 | ★★★★★ |
| 品类创新 | 放置战棋新品类，先发优势 | ★★★★☆ |
| 玩法融合 | 三种玩法有机结合，差异化明显 | ★★★★☆ |
| 轻度体验 | 适合移动端碎片时间 | ★★★★☆ |

## 八、后续工作建议

### 8.1 下一阶段优先事项

| 优先级 | 工作内容 | 产出目标 |
|-------|---------|---------|
| P0 | 战斗引擎核心数值表 | 定义所有基础数值 |
| P0 | 首批10名武将技能设计 | 验证技能战棋化方案 |

| P1 | UI原型设计 | 核心界面布局 |
| P2 | 经济系统数值调参 | 资源产出/消耗表 |
| P2 | 关卡编辑器设计 | 快速制作关卡 |

### 8.2 需要深入研究的方向

1. **战斗数值平衡**：各品质武将的基础属性、技能伤害系数、怒气消耗的精确平衡
2. **完整武将池设计**：首批40名武将的详细设计文档，包括技能效果、数值、羁绊
3. **PVP竞技系统**：匹配算法、段位设计、Ban/Pick机制
4. **关卡编辑器**：让玩家自制关卡，增加UGC内容

### 8.3 潜在风险与应对

| 风险 | 描述 | 应对方案 |
|-----|------|---------|
| 开发延期 | 技能系统复杂可能影响进度 | 砍非核心功能，保证核心循环 |
| 市场不认可 | 放置战棋概念可能不被接受 | 强调"懒人战棋"等易懂概念 |
| 数值失控 | 多系统耦合导致数值膨胀 | 建立数值上限，赛季重置 |
| IP纠纷 | 三国杀IP授权可能受限 | 提前沟通，建立备选方案 |
