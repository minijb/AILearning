## 三、优先级路线图

### 3.1 第一个月（MVP 发布）

**目标：** 验证核心循环，建立最小可玩版本

**功能清单：**
- 战斗引擎（格子、移动、攻击、伤害）
- 放置系统（拖拽、阵型）
- 技能系统（主动、被动、怒气）
- 关卡系统（3关、教学关）
- 武将系统（4名初始武将、升级）
- 抽卡系统（单抽、十连、保底）
- 基础资源系统

**验收标准：**
- [ ] 能完成一局完整战斗
- [ ] 能放置武将并观看自动战斗
- [ ] 能释放手动技能
- [ ] 能获得武将并升级
- [ ] 能抽取新武将

### 3.2 第二至三个月（正式版本）

**目标：** 完善成长系统，提供长线内容

**功能清单：**
- L1 全部功能完善
- 装备系统（穿戴、强化、套装）
- 阵法系统
- 羁绊系统
- 军衔系统
- 技能升级/突破
- 关卡扩展至第一章（12关）
- 精英关卡
- 成就系统

**验收标准：**
- [ ] 玩家能体验完整的养成循环
- [ ] 章节内容足够支撑2周体验
- [ ] 日常任务和周常任务可用
- [ ] 装备系统有足够深度

### 3.3 第四至六个月（内容扩展）

**目标：** 增加策略深度，提高重复可玩性

**功能清单：**
- 天赋系统
- 觉醒系统
- 噩梦/地狱难度
- 无尽模式
- 词缀系统
- 第二章内容（12关）
- 联盟系统（基础版）
- 竞技系统（基础版）

**验收标准：**
- [ ] 硬核玩家有足够挑战内容
- [ ] 社交功能开始运转
- [ ] PVP内容可用
- [ ] 第二章支撑2周体验

### 3.4 第七至十二个月（商业化完善）

**目标：** 完善商业化，建立长期留存

**功能清单：**
- 联盟系统（完整版）
- 竞技系统（完整版）
- 赛季系统
- 皮肤系统
- 战令系统
- 赛季3-4章内容
- 关卡编辑器（可选）
- 观战系统（可选）

**验收标准：**
- [ ] 月流水达到目标
- [ ] 用户留存数据健康
- [ ] 付费率在预期范围
- [ ] 社区生态初步形成
