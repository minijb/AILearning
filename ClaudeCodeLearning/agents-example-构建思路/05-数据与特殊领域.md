## 四、database-design — 数据库设计

### 4.1 核心 Agent

| Agent | 职责 |
|-------|------|
| `database-architect` | 数据库架构师 |

### 4.2 设计原则

```markdown
## 数据库设计模式

1. **规范化 vs 反规范化**
   - OLTP: 3NF/BCNF
   - OLAP: 维度建模、星型/雪花型

2. **索引策略**
   - B-tree: 范围查询、排序
   - Hash: 等值查询
   - GIN/GiST: 全文搜索、JSON

3. **分区策略**
   - Range: 时间序列数据
   - Hash: 均匀分布
   - List: 离散值
```

## 二十三、核心设计亮点

### 23.1 领域垂直深度

blockchain-web3 的 4 个 Skill 覆盖了从 DeFi 协议到 NFT 标准到智能合约安全的完整 Web3 开发栈。quantitative-trading 专注于金融量化领域。payment-processing 深入 Stripe/PayPal/PCI 合规。这种垂直深度是 agents_example 的核心价值。

### 23.2 C4 模型的 Agent 化

C4 架构是一个 4 层文档系统，在 agents_example 中被实现为 4 个独立 Agent，每个 Agent 负责一个层级的架构图生成。这种对应关系非常自然：C4 层 → Agent。

### 23.3 游戏开发的 niche 覆盖

Unity ECS 和 Godot GDScript 是游戏开发领域的两个主流技术栈，尽管这个插件很小众，但体现了 agents_example "只要有需求就有插件"的设计哲学。
