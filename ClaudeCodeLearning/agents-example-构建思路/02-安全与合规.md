# agents_example 插件分类分析 — 安全与合规

## 一、security-scanning — 安全扫描

### 1.1 插件概述

安全扫描插件是 agents_example 中**流程最完整**的编排命令之一，提供从漏洞扫描到报告生成的全链路安全评估。

### 1.2 核心 Agent

| Agent | 职责 |
|-------|------|
| `security-auditor` | 综合安全评估 |

### 1.3 安全强化编排器（security-hardening）

**设计亮点**：严格阶段门控 + 状态持久化

```markdown
## CRITICAL BEHAVIORAL RULES
1. Execute steps in order. Do NOT skip ahead.
2. Write output files. Each step MUST produce output.
3. Stop at PHASE CHECKPOINT. Wait for user approval.
4. Halt on failure. STOP immediately.
5. Use only local agents. No cross-plugin dependencies.
6. Never enter plan mode autonomously.
```

**6 阶段工作流**：

| 阶段 | 内容 | 步骤 |
|------|------|------|
| Phase 1 | 评估与威胁建模 | 漏洞扫描、威胁建模 |
| Phase 2 | SAST 静态分析 | 自动化 SAST 扫描 |
| Phase 3 | 依赖扫描 | 已知漏洞检测 |
| Phase 4 | 代码审查 | 认证、授权、加密、注入 |
| Phase 5 | 配置审计 | Infrastructure as Code |
| Phase 6 | 报告生成 | 发现汇总 + 修复建议 |

### 1.4 支持的合规框架

```
--compliance owasp,soc2,gdpr,hipaa,pci-dss
```

**OWASP Top 10**：
- A01: Broken Access Control
- A02: Cryptographic Failures
- A03: Injection
- A04: Insecure Design
- A05: Security Misconfiguration
- A06: Vulnerable Components
- A07: Auth Failures
- A08: Data Integrity Failures
- A09: Logging Failures
- A10: SSRF

### 1.5 状态持久化

```json
// .security-hardening/state.json
{
  "target": "$ARGUMENTS",
  "status": "in_progress",
  "depth": "comprehensive",
  "compliance_frameworks": ["owasp"],
  "current_step": 1,
  "current_phase": 1,
  "completed_steps": [],
  "files_created": [],
  "started_at": "ISO_TIMESTAMP",
  "last_updated": "ISO_TIMESTAMP"
}
```

支持**会话恢复**：
- 状态为 `in_progress` → 询问恢复或重新开始
- 状态为 `complete` → 询问归档或重新开始

## 七、核心设计亮点

### 7.1 阶段门控 + 状态持久化

安全扫描插件的工作流设计是整个 agents_example 中**最严格的**：
- 每个阶段必须产生输出文件
- `PHASE CHECKPOINT` 强制用户确认
- 失败立即停止，不静默继续
- 状态文件支持会话恢复

### 7.2 深度分层策略

| 深度 | 覆盖范围 |
|------|---------|
| `quick` | 快速扫描，基础安全检查 |
| `standard` | 标准扫描，OWASP Top 10 |
| `comprehensive` | 全面扫描，所有合规框架 |

### 7.3 本地 Agent 约束

> "Use only local agents. No cross-plugin dependencies."

这确保安全扫描不依赖外部插件，在隔离环境中可靠运行。
