## 11. 字符串处理

### 11.1 字符串基础操作

#### 获取字符串长度

```bash
str="Hello World"
echo ${#str}          # 输出: 11
```

#### 字符串切片（Substring）

```bash
str="Hello World"

# 从位置2开始，截取5个字符（索引从0开始）
echo ${str:2:5}        # 输出: llo W

# 从末尾开始截取
echo ${str: -5}        # 输出: World（注意空格）
echo ${str: -5:3}      # 输出: Wor
```

### 11.2 字符串删除与替换

#### 删除子串

```bash
str="Hello World"

# 删除最短匹配的前缀
echo ${str#*o}         # 输出: World（删除"Hel"）

# 删除最长匹配的前缀
echo ${str##*o}        # 输出: rld

# 删除最短匹配的后缀
echo ${str%o*}         # 输出: Hello W

# 删除最长匹配的后缀
echo ${str%%o*}        # 输出: Hell
```

#### 字符串替换

```bash
str="Hello World"

# 替换第一个匹配
echo ${str/World/Linux}    # 输出: Hello Linux

# 替换所有匹配
echo ${str//o/O}           # 输出: HellO WOrld

# 从开头匹配替换
echo ${str/#Hello/Hi}     # 输出: Hi World

# 从末尾匹配替换
echo ${str/%World/Linux}   # 输出: Hello Linux

# 替换为空（删除）
echo ${str//o/}           # 输出: Hell Wrld
```

### 11.3 字符串大小写转换（Bash 4.0+）

```bash
str="Hello World"

echo ${str,,}             # 小写: hello world
echo ${str^^}             # 大写: HELLO WORLD
echo ${str,,[A-Z]}        # 首字母大写: Hello world
echo ${str^^[a-z]}        # 首字母大写: Hello World
```

### 11.4 expr 命令

`expr` 是古老的表达式计算工具，也可用于字符串操作：

```bash
# 计算字符串长度
expr length "Hello"              # 输出: 5

# 查找子串位置（从1开始）
expr index "Hello World" "Wor"   # 输出: 7

# 提取子串
expr substr "Hello World" 1 5     # 输出: Hello

# 正则匹配
expr match "Hello123" '.*\([0-9]\+\)'   # 输出: 123
```

### 11.5 cut 命令

按列剪切文本：

```bash
# 按字符位置剪切
echo "Hello World" | cut -c1-5      # 输出: Hello

# 按字节剪切（多字节字符）
echo "你好世界" | cut -c1-3         # 输出: 你

# 按分隔符剪切
echo "a,b,c,d" | cut -d',' -f1,3    # 输出: a,c
echo "a,b,c,d" | cut -d',' -f2-3    # 输出: b,c

# 剪切多个字段
echo "root:x:0:0:Admin:/root:/bin/bash" | cut -d':' -f1,6
# 输出: root:/root
```

### 11.6 tr 命令

字符转换和删除：

```bash
# 大小写转换
echo "Hello World" | tr 'a-z' 'A-Z'    # 输出: HELLO WORLD
echo "HELLO" | tr 'A-Z' 'a-z'          # 输出: hello

# 删除指定字符
echo "Hello123World456" | tr -d '0-9'   # 输出: HelloWorld

# 压缩重复字符
echo "Hellooo    World" | tr -s ' '     # 输出: Hellooo World
echo "aaabbbccc" | tr -s 'ab'           # 输出: abccc

# 字符集操作
echo "Hello123" | tr -c 'a-zA-Z' '\n'  # 替换非字母为换行
```

## 12. 数组

### 12.1 普通数组

#### 定义数组

```bash
# 方式1：直接赋值
arr=(one two three four)

# 方式2：按索引赋值
arr[0]="apple"
arr[1]="banana"
arr[2]="orange"

# 方式3：从命令输出创建
arr=($(ls *.txt))           # 将所有txt文件放入数组

# 方式4：定义时指定索引
arr=([0]="apple" [5]="banana" [2]="orange")
```

#### 访问数组元素

```bash
arr=(apple banana orange)

echo ${arr[0]}              # 输出: apple
echo ${arr[2]}              # 输出: orange
echo ${arr[@]}              # 输出: apple banana orange（所有元素）
echo ${arr[*]}              # 输出: apple banana orange（所有元素）
echo ${#arr[@]}             # 输出: 3（数组长度）
echo ${#arr[0]}             # 输出: 5（第一个元素的长度）
```

#### 数组切片

```bash
arr=(a b c d e f g)

echo ${arr[@]:1:3}          # 输出: b c d（从索引1开始，取3个）
echo ${arr[@]: -3:2}        # 输出: e f（从末尾第3个开始，取2个）
```

#### 数组追加

```bash
arr=(one two)
arr+=(three four)           # 追加多个元素
echo ${arr[@]}              # 输出: one two three four
```

#### 删除数组元素

```bash
arr=(a b c d)
unset arr[1]                # 删除索引1的元素
echo ${arr[@]}              # 输出: a c d

unset arr                   # 删除整个数组
```

### 12.2 关联数组（Bash 4.0+）

关联数组使用字符串作为索引：

```bash
# 必须先声明
declare -A person

# 赋值
person["name"]="张三"
person["age"]=25
person["city"]="北京"

# 访问
echo ${person["name"]}      # 输出: 张三
echo ${person[@]}            # 输出: 张三 25 北京（所有值）
echo ${!person[@]}           # 输出: name age city（所有键）
echo ${#person[@]}           # 输出: 3（元素个数）
```

#### 关联数组示例：遍历

```bash
declare -A config
config["host"]="localhost"
config["port"]="3306"
config["dbname"]="mydb"

# 遍历键值对
for key in "${!config[@]}"; do
    echo "$key = ${config[$key]}"
done
```

### 12.3 数组操作技巧

#### 数组去重

```bash
arr=(a b a c b d a)
unique=($(echo "${arr[@]}" | tr ' ' '\n' | sort -u | tr '\n' ' '))
echo "${unique[@]}"          # 输出: a b c d
```

#### 数组排序

```bash
arr=(banana apple cherry date)
sorted=($(for i in "${arr[@]}"; do echo $i; done | sort))
echo "${sorted[@]}"          # 输出: apple banana cherry date
```

#### 查找元素索引

```bash
arr=(foo bar baz qux)

for i in "${!arr[@]}"; do
    if [[ "${arr[$i]}" == "bar" ]]; then
        echo "Found at index $i"
    fi
done
```

#### 二维数组模拟

```bash
# Bash 不支持真正的二维数组，但可以用一维数组模拟
declare -A matrix
rows=2
cols=3

for i in $(seq 0 $((rows-1))); do
    for j in $(seq 0 $((cols-1))); do
        idx=$((i * cols + j))
        matrix[$i,$j]=$((idx + 1))
    done
done

# 访问元素
echo ${matrix[1,2]}          # 输出: 6
```

## 13. 正则表达式

### 13.1 基本正则（BRE）与扩展正则（ERE）

```bash
# 基本正则（BRE）- grep 默认
grep 'foo.*bar' file.txt

# 扩展正则（ERE）- grep -E 或 egrep
grep -E 'foo|bar' file.txt
egrep 'foo|bar' file.txt

# 使用 grep -P 启用 Perl 正则（如果支持）
grep -P '\d+' file.txt
```

### 13.2 正则表达式元字符

| 元字符 | 含义 | 示例 |
|--------|------|------|
| `.` | 任意单个字符 | `a.c` 匹配 "abc" |
| `^` | 行开头 | `^Hello` 匹配行首的 Hello |
| `$` | 行结尾 | `World$` 匹配行尾的 World |
| `*` | 零个或多个 | `ab*` 匹配 "a", "ab", "abb" |
| `+` | 一个或多个（ERE） | `ab+` 匹配 "ab", "abb" |
| `?` | 零个或一个（ERE） | `colou?r` 匹配 "color", "colour" |
| `[...]` | 字符集 | `[a-z]` 小写字母 |
| `[^...]` | 否定字符集 | `[^0-9]` 非数字 |
| `{n}` | 精确n次 | `a{3}` 匹配 "aaa" |
| `{n,}` | n次或更多 | `a{2,}` 匹配 "aa" 以上 |
| `{n,m}` | n到m次 | `a{2,4}` 匹配 2-4 个 a |
| `()` | 分组（ERE） | `(ab)+` 匹配 "ab", "abab" |
| `\|` | 或（ERE） | `cat\|dog` 匹配 "cat" 或 "dog" |
| `\` | 转义 | `\.` 匹配字面句号 |

### 13.3 grep 中的正则

```bash
# 基本匹配
grep 'error' /var/log/syslog

# 正则匹配
grep '^202[0-9]' access.log           # 以2020-2029开头的行
grep '[0-9]\{3\}\.[0-9]\{3\}' log.txt # IP地址格式
grep -E '\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b' emails.txt  # 邮箱

# 统计匹配行数
grep -c 'error' log.txt

# 显示匹配的行号
grep -n 'pattern' file.txt

# 显示不匹配的行
grep -v 'pattern' file.txt

# 显示上下文
grep -A 2 'pattern' file.txt          # 匹配行后2行
grep -B 2 'pattern' file.txt          # 匹配行前2行
grep -C 2 'pattern' file.txt          # 前后各2行

# 递归搜索
grep -r 'pattern' /path/to/dir

# 只显示匹配的部分
grep -o 'pattern' file.txt

# 忽略大小写
grep -i 'pattern' file.txt
```

### 13.4 sed 中的正则

```bash
# 替换
sed -E 's/([0-9]+)/[\1]/g' file.txt           # 数字加方括号
sed -E 's/\b([a-z]+)\b/\1\1/g' file.txt       # 重复单词

# 删除匹配行
sed '/^#/d' file.txt                          # 删除注释行
sed '/^$/d' file.txt                           # 删除空行
sed '/^\s*#/d' file.txt                        # 删除空白开头的注释行

# 保留匹配行
sed -n '/pattern/p' file.txt

# 多重替换
sed -e 's/foo/bar/' -e 's/baz/qux/' file.txt

# 使用 & 引用匹配内容
echo "abc" | sed 's/.*/"&"/'                   # 输出: "abc"
echo "123" | sed 's/[0-9]/(&)/'                 # 输出: (123)

# 捕获组
echo "2024-03-15" | sed -E 's/([0-9]{4})-([0-9]{2})-([0-9]{2})/\3\/\2\/\1/'
# 输出: 15/03/2024
```

### 13.5 awk 中的正则

```bash
# 正则作为条件
awk '/^202[0-9]/ {print}' file.txt

# 字段匹配
awk -F: '$1 ~ /^root/ {print}' /etc/passwd

# 否定匹配
awk '$0 !~ /error/ {print}' file.txt

# 正则范围
awk '/start/,/end/ {print}' file.txt          # 打印 start 到 end 之间的行

# 正则内置函数
awk '{
    if (match($0, /[0-9]+/)) {
        print "Found number at position", RSTART, "length", RLENGTH
    }
}' file.txt

# sub/gsub 替换
awk '{sub(/old/, "new"); print}' file.txt
awk '{gsub(/[0-9]+/, "NUM"); print}' file.txt

# split 函数
awk '{n=split($0, arr, /[:\/.]/); for(i=1;i<=n;i++) print arr[i]}' file.txt
```

## 14. 命令替换与进程管理

### 14.1 命令替换

#### 反引号语法

```bash
current_date=`date +%Y-%m-%d`
echo "今天是: $current_date"

# 嵌套使用
result=`echo "当前目录: `pwd`"`
```

#### $() 语法（推荐）

```bash
current_date=$(date +%Y-%m-%d)
echo "今天是: $current_date"

# 可嵌套，易读
result=$(echo "Home: $(echo ~)")
```

### 14.2 进程管理基础

#### 后台运行

```bash
# 使用 & 将命令放到后台
long_running_command &

# jobs 查看后台任务
jobs
# 输出: [1]+ Running long_running_command &

# fg 将任务放到前台
fg %1

# bg 将挂起的任务在后台继续运行
bg %1
```

#### nohup 与 disown

```bash
# nohup - 忽略挂断信号，关闭终端后继续运行
nohup ./script.sh > output.log 2>&1 &

# disown - 将运行中的任务从当前 shell 分离
./script.sh &
disown %1
disown -h %1    # 标记不过滤 SIGHUP
disown -a       # 分离所有任务
disown -r       # 只分离正在运行的
```

#### ps 命令

```bash
# 查看当前终端的进程
ps

# 查看所有进程
ps aux
ps -ef

# 显示进程树
ps auxf
ps -ef --forest

# 按用户过滤
ps -u username

# 按进程名过滤
ps -C nginx,apache2

# 查看特定进程
ps aux | grep nginx

# 格式化输出
ps -o pid,ppid,cmd --sort=-cpu
```

#### kill 命令

```bash
# 优雅终止（SIGTERM，默认）
kill 1234

# 强制终止（SIGKILL，无法捕获）
kill -9 1234
kill -KILL 1234

# 发送其他信号
kill -HUP 1234    # 重载配置
kill -STOP 1234   # 暂停进程
kill -CONT 1234   # 继续进程

# 按名称杀进程
pkill nginx
killall nginx

# 只杀自己的进程
pkill -u $USER nginx
```

#### 进程监控

```bash
# top - 实时监控
top
top -u username        # 只显示指定用户
top -p 1234,5678      # 只显示指定PID
top -d 2              # 每2秒刷新

# htop - 更友好的 top（需安装）
htop

# 查看特定端口
lsof -i :8080
ss -tlnp | grep :8080

# 查看进程打开的文件
lsof -p 1234
```

### 14.3 wait 命令

等待后台任务完成：

```bash
# 等待所有后台任务
sleep 5 &
sleep 3 &
wait
echo "所有后台任务完成"

# 等待特定后台任务
sleep 5 &
PID=$!
sleep 3
wait $PID
echo "任务 $PID 完成"
```

### 14.4 进程替换

```bash
# 替换文件参数
diff <(ls /bin) <(ls /usr/bin)

# 替换文件描述符
while read line; do
    echo "Line: $line"
done < <(grep pattern file.txt)
```

### 14.5 协程与 coproc

```bash
# 使用 coproc 创建协程
coproc MYPROC { while read line; do echo "Got: $line"; done; }

# 发送数据
echo "hello" >&${MYPROC[1]}
echo "world" >&${MYPROC[1]}

# 读取输出
cat <&${MYPROC[0]}

# 关闭写入端
exec ${MYPROC[1]}>&-

# 等待协程结束
wait $MYPROC_PID
```

## 15. 调试与错误处理

### 15.1 set 命令选项

#### 常用选项

```bash
#!/bin/bash

# -x: 打印命令及参数（调试模式）
set -x

# -e: 遇到错误立即退出
set -e

# -u: 使用未定义的变量时报错
set -u

# -n: 读取命令但不执行（语法检查）
set -n

# -v: 打印原始输入行
set -v

# -f: 禁用文件名通配符展开
set -f
```

#### 组合使用

```bash
#!/bin/bash

# 严格模式（推荐）
set -euo pipefail

# -e: 脚本中的任何命令失败（非0退出码）立即退出
# -u: 引用未定义变量时报错
# -o pipefail: 管道中任何一个命令失败，整个管道返回失败码
```

### 15.2 调试技巧

#### 方法1：在脚本中启用

```bash
#!/bin/bash

# 启用调试
set -x

echo "Debug mode started"
name="World"
echo "Hello $name"

# 禁用调试
set +x

echo "Debug mode ended"
```

#### 方法2：命令行启用

```bash
# 调试执行
bash -x script.sh

# 语法检查
bash -n script.sh

# 调试并跟踪错误
bash -x -e script.sh
```

#### 方法3：DEBUG 信号

```bash
#!/bin/bash

# 在每个命令执行前触发 DEBUG
trap 'echo "Executing: $BASH_COMMAND"' DEBUG

# 跟踪变量
trap 'echo "VAR=$VAR"' DEBUG

echo "test"
VAR=123
```

### 15.3 trap 信号处理

#### 基本语法

```bash
trap 'commands' signals
trap "commands" signals
trap 'commands' SIGTERM SIGINT
```

#### 常用信号

| 信号 | 编号 | 说明 |
|------|------|------|
| SIGHUP | 1 | 挂起 |
| SIGINT | 2 | Ctrl+C |
| SIGQUIT | 3 | Ctrl+\ |
| SIGTERM | 15 | 优雅终止 |
| EXIT | 0 | 脚本退出 |

#### 清理临时文件

```bash
#!/bin/bash

tmpfile1=$(mktemp)
tmpfile2=$(mktemp)

# 脚本退出时清理
trap 'rm -f "$tmpfile1" "$tmpfile2"' EXIT

# 也可以分开处理
trap 'rm -f "$tmpfile1" "$tmpfile2"; exit 1' SIGINT SIGTERM
```

#### 忽略信号

```bash
#!/bin/bash

# 忽略 Ctrl+C
trap '' SIGINT
echo "按 Ctrl+C 无效，按 Ctrl+\\ 退出"

# 恢复信号
trap SIGINT
echo "现在 Ctrl+C 可以正常工作了"
```

#### 调试和日志

```bash
#!/bin/bash

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $*" >&2
}

# 记录函数调用
trace_calls() {
    log "Calling: ${FUNCNAME[1]} with args: $*"
}

trap 'log "Script exited with code $?"' EXIT
trap 'log "Received SIGINT, exiting..."' SIGINT

log "Starting script"
```

### 15.4 错误处理函数

#### 简单错误处理

```bash
#!/bin/bash

set -e

error_exit() {
    echo "ERROR: $1" >&2
    exit 1
}

# 使用 || 简化
command1 || error_exit "command1 failed"
command2 || error_exit "command2 failed"

# 或者用 &&
ls /tmp || echo "Failed to list /tmp"
```

#### 高级错误处理

```bash
#!/bin/bash

set -euo pipefail

errexit() {
    local err=$?
    local line_no=$1
    echo "Error on line $line_no (exit code: $err)" >&2
    exit $err
}

trap 'errexit $LINENO' ERR

# 当任何命令失败时会自动触发 ERR
command1
command2
```

### 15.5 条件测试与返回码

```bash
#!/bin/bash

    echo "备份成功"

```

### 15.6 调试脚本模板

```bash
#!/bin/bash
#
# 调试模板脚本
# 使用方法: bash -x debug_template.sh
#

# 严格模式
set -euo pipefail

# 调试开关
DEBUG=${DEBUG:-false}
TRACE=${TRACE:-false}

# 日志函数
log_info() { echo "[INFO] $(date '+%H:%M:%S') $*"; }
log_err()  { echo "[ERROR] $(date '+%H:%M:%S') $*" >&2; }
log_debug(){ $DEBUG && echo "[DEBUG] $(date '+%H:%M:%S') $*" || true; }

# 调试模式下启用 xtrace
$TRACE && set -x

# 清理函数
cleanup() {
    local exit_code=$?
    log_debug "Cleaning up..."
    # 添加清理代码
    exit $exit_code
}
trap cleanup EXIT

# 主函数
main() {
    log_info "脚本开始执行"

    # 主逻辑
    log_info "执行完成"
}

main "$@"
```

### 15.7 ShellCheck 集成

ShellCheck 是一个静态分析工具，能检测 shell 脚本中的常见错误：

```bash
# 安装
# Ubuntu/Debian: sudo apt install shellcheck
# macOS: brew install shellcheck

# 使用
shellcheck script.sh

# 检查并输出详细信息
shellcheck -x -e SC1090 script.sh

# 常见选项
# -x: 跟踪 sourced 文件
# -e: 排除特定警告
# -f: 输出格式 (text, gcc, checkstyle, json)
shellcheck -f json script.sh > issues.json
```
