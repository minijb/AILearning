## 6. 条件判断

### 6.1 条件测试的三种形式

Bash 中有三种条件测试语法，本质上都是调用 `test` 命令：

```bash
# 形式一：test 命令（最传统）
test "$age" -ge 18 && echo "成年"

# 形式二：单方括号 [ ]（最常用，需注意空格）
[ "$age" -ge 18 ] && echo "成年"

# 形式三：双方括号 [[ ]]（推荐，支持更多特性）
[[ "$age" -ge 18 ]] && echo "成年"
```

> **注意**：`[` 本身是一个命令，不是语法，因此左右括号与内容之间必须留有空格。

### 6.2 文件测试运算符

判断文件属性的运算符是脚本中最常用的条件之一：

| 运算符 | 含义 |
|--------|------|
| `-e file` | 文件是否存在 |
| `-f file` | 是否为普通文件 |
| `-d file` | 是否为目录 |
| `-r file` | 是否可读 |
| `-w file` | 是否可写 |
| `-x file` | 是否可执行 |
| `-L file` | 是否为符号链接 |
| `-s file` | 文件是否存在且非空 |
| `file1 -nt file2` | file1 是否比 file2 更新（newer than） |
| `file1 -ot file2` | file1 是否比 file2 更旧（older than） |

**示例：检查文件是否存在并可执行**

```bash
#!/bin/bash
FILE="/usr/local/bin/myapp"

if [ -f "$FILE" ]; then
    if [ -x "$FILE" ]; then
        echo "$FILE 存在且可执行"
    else
        echo "$FILE 存在但不可执行，添加执行权限..."
        chmod +x "$FILE"
    fi
else
    echo "$FILE 不存在"
fi
```

### 6.3 字符串比较

| 运算符 | 含义 |
|--------|------|
| `-z string` | 字符串长度为 0（空串） |
| `-n string` | 字符串长度非 0（非空） |
| `str1 = str2` | 字符串相等 |
| `str1 != str2` | 字符串不相等 |
| `str1 < str2` | str1 字典序小于 str2 |
| `str1 > str2` | str1 字典序大于 str2 |

**重要原则**：变量值用双引号包裹后再比较，避免空值导致语法错误。

```bash
# 推荐写法
if [ "$name" = "admin" ]; then

# 可能出错的写法（name为空时变为 [ = "admin" ]）
if [ $name = "admin" ]; then
```

**[[ ]] 的优势**：无需引号包裹变量，支持 `&&` 和 `||` 运算符：

```bash
name="alice"

# 单括号写法（需分开写）
[ "$name" = "alice" ] && [ "$name" != "bob" ]

# 双方括号写法（更直观）
[[ "$name" = "alice" && "$name" != "bob" ]]
```

### 6.4 数字比较

| 运算符 | 含义 |
|--------|------|
| `-eq` | 等于（equal） |
| `-ne` | 不等于（not equal） |
| `-gt` | 大于（greater than） |
| `-ge` | 大于等于（greater or equal） |
| `-lt` | 小于（less than） |
| `-le` | 小于等于（less or equal） |

```bash
score=85

if [ "$score" -ge 90 ]; then
    echo "优秀"
elif [ "$score" -ge 60 ]; then
    echo "及格"
else
    echo "不及格"
fi
```

> **注意**：这些运算符是用于整数的，不能用于小数比较。小数比较需要使用 `bc` 或 `awk`。

### 6.5 复合条件

```bash
# AND 逻辑：两个条件都必须满足
if [ "$age" -ge 18 ] && [ "$has_license" = "yes" ]; then
    echo "可以开车"
fi

# OR 逻辑：满足任一条件即可
if [ "$role" = "admin" ] || [ "$role" = "manager" ]; then
    echo "有管理权限"
fi

# NOT 逻辑：取反
if [ ! -f "$file" ]; then
    echo "文件不存在"
fi
```

## 7. 循环结构

### 7.1 for 循环

**基本语法 — 遍历列表**

```bash
for 变量 in 列表; do
    # 使用 $变量
done
```

```bash
# 遍历字符串列表
for color in red green blue; do
    echo "颜色: $color"
done

# 遍历命令输出
for file in $(ls *.txt); do
    echo "文本文件: $file"
done

# 遍历数字序列（seq 命令或 {start..end}）
for i in {1..5}; do
    echo "第 $i 次"
done

# 遍历数组
fruits=(apple banana cherry)
for fruit in "${fruits[@]}"; do
    echo "水果: $fruit"
done
```

**C 语言风格 for 循环**

Bash 3.0+ 支持类似 C 的 for 循环语法：

```bash
for ((i = 0; i < 5; i++)); do
    echo "计数器: $i"
done

# 结合变量
limit=10
for ((i = 1; i <= limit; i++)); do
    echo "$i"
done
```

### 7.2 while 循环

条件为真时持续执行，适合处理不确定次数的循环：

```bash
while 条件; do
    # 循环体
done
```

```bash
# 逐行读取文件
while read -r line; do
    echo "行内容: $line"
done < "input.txt"

# 计数器循环
count=1
while [ "$count" -le 5 ]; do
    echo "计数: $count"
    ((count++))
done

# 读取用户输入直到退出
while true; do
    read -p "输入命令（q退出）: " cmd
    if [ "$cmd" = "q" ]; then
        break
    fi
    echo "执行: $cmd"
done
```

### 7.3 until 循环

与 while 相反：条件为**假**时持续执行，直到条件变为真：

```bash
# 等待服务启动（服务就绪前一直等待）
until curl -s http://localhost:8080 > /dev/null; do
    echo "等待服务启动..."
    sleep 2
done
echo "服务已就绪"

# 等到文件出现
until [ -f "/tmp/ready" ]; do
    sleep 1
done
echo "就绪文件已创建"
```

### 7.4 循环控制：break 与 continue

```bash
# break：跳出整个循环
for i in {1..10}; do
    if [ "$i" -eq 5 ]; then
        echo "遇到 5，退出循环"
        break
    fi
    echo "处理 $i"
done

# continue：跳过本次迭代，进入下一次
for i in {1..5}; do
    if [ "$i" -eq 3 ]; then
        echo "跳过 3"
        continue
    fi
    echo "处理 $i"
done
# 输出：1, 2, 跳过 3, 4, 5
```

### 7.5 循环读取文件

最稳健的逐行读取方式：

```bash
# 方法一：while read（推荐）
while IFS= read -r line; do
    echo "$line"
done < "file.txt"

# IFS= 防止首尾空格被吃掉
# -r 防止反斜杠被解释

# 方法二：处理含特殊字符的文件名
while IFS= read -r -d '' line; do
    echo "$line"
done < <(find . -type f -print0)
```

## 8. 分支结构

### 8.1 if / elif / else 语句

```bash
if 条件1; then
    # 条件1为真时执行
elif 条件2; then
    # 条件2为真时执行
else
    # 所有条件都不满足时执行
fi
```

```bash
#!/bin/bash
grade=78

if [ "$grade" -ge 90 ]; then
    echo "等级: A"
elif [ "$grade" -ge 80 ]; then
    echo "等级: B"
elif [ "$grade" -ge 70 ]; then
    echo "等级: C"
elif [ "$grade" -ge 60 ]; then
    echo "等级: D"
else
    echo "等级: F（不及格）"
fi
```

**单行简写形式**

```bash
# 三元表达式风格
[ "$age" -ge 18 ] && echo "成年" || echo "未成年"

# 等价于
if [ "$age" -ge 18 ]; then
    echo "成年"
else
    echo "未成年"
fi
```

### 8.2 case 语句

当需要对一个变量与多个模式匹配时，`case` 比多个 `if/elif` 更清晰：

```bash
case 变量 in
    模式1)
        # 匹配模式1时执行
        ;;
    模式2|模式3)
        # 匹配模式2或模式3
        ;;
    *)
        # 默认匹配（类似 else）
        ;;
esac
```

```bash
#!/bin/bash
read -p "你喜欢什么动物？ " animal

case "$animal" in
    dog)
        echo "狗是人类最好的朋友"
        ;;
    cat)
        echo "猫很高冷但很可爱"
        ;;
    bird|parrot)
        echo "鸟类很聪明"
        ;;
    fish)
        echo "金鱼记忆只有7秒"
        ;;
    *)
        echo "我没见过这种动物"
        ;;
esac
```

**常见模式匹配符号**

```bash
case "$filename" in
    *.txt)        echo "文本文件" ;;
    *.jpg|*.png)  echo "图片文件" ;;
    temp*)        echo "临时文件" ;;
    ??)           echo "两个字符的文件名" ;;
    [0-9]*)       echo "以数字开头的文件名" ;;
    *)            echo "其他文件" ;;
esac
```

## 9. 位置参数与特殊变量

### 9.1 位置参数

执行脚本时，命令行上传递的参数按顺序赋值给 `$1`, `$2`, `$3`...：

```bash
#!/bin/bash
# 脚本名：greet.sh
echo "脚本名: $0"
echo "第一个参数: $1"
echo "第二个参数: $2"
echo "第三个参数: $3"
```

```bash
$ bash greet.sh Alice Bob Charlie
脚本名: greet.sh
第一个参数: Alice
第二个参数: Bob
第三个参数: Charlie
```

### 9.2 特殊变量汇总

| 变量 | 含义 |
|------|------|
| `$0` | 脚本名称（包含路径） |
| `$1` ~ `$9` | 第1~9个参数 |
| `${10}` | 第10个及以上参数（需用大括号） |
| `$#` | 参数个数 |
| `$@` | 所有参数（各自独立，作为词列表） |
| `$*` | 所有参数（作为单一字符串） |
| `$?` | 上一个命令的退出状态（0=成功） |
| `$$` | 当前脚本的进程 ID |
| `$!` | 最近后台任务的进程 ID |
| `$-` | 当前设置的 shell 选项（如 `himBH`） |

```bash
#!/bin/bash
echo "参数个数: $#"
echo "所有参数(*): $*"
echo "所有参数(@): $@"
echo "上一条命令退出状态: $?"

# 注意 $* 和 $@ 的区别（见下方 9.4）
```

### 9.3 shift 命令

`shift` 将参数列表**左移一位**，使所有位置参数向前移动一位，常用于遍历未知数量的参数。

**图示：shift 的具体效果**

假设执行脚本时传入 4 个参数：

```bash
./script.sh a b c d
```

调用 `shift` 后，参数变化如下：

```
初始状态：
$1=a  $2=b  $3=c  $4=d   $#=4

执行一次 shift（左移一位）：
$1=b  $2=c  $3=d         $#=3   ← $1 从 a 变成 b，$# 减 1

执行两次 shift（再左移一位）：
$1=c  $2=d               $#=2   ← $1 从 b 变成 c，$# 再减 1
```

> **形象理解**：把参数想象成队伍，`shift` 就是"最前面的人出列"，后面的人全部向前补位。执行 N 次 `shift` 后，原来第 N+1 个参数就变成了新的 `$1`。

**与 while 循环配合：逐个处理所有参数**

这是 `shift` 最常见的用法——在不知道参数个数的情况下，依次处理每一个：

```bash
#!/bin/bash
echo "共 $# 个文件"

while [ "$#" -gt 0 ]; do
    echo "处理文件: $1"
    shift  # 每次左移一位，下一个参数变成新的 $1
done
```

```bash
$ bash process.sh file1.txt file2.txt file3.txt
共 3 个文件
处理文件: file1.txt
处理文件: file2.txt
处理文件: file3.txt
```

**实战场景：处理文件名参数**

```bash
#!/bin/bash
# 将所有传入的文件复制到 /backup 目录
while [ "$#" -gt 0 ]; do
    if [ -f "$1" ]; then
        cp "$1" /backup/
        echo "已备份: $1"
    else
        echo "警告: $1 不是普通文件，跳过"
    fi
    shift
done
echo "备份完成！"
```

### 9.4 "$@" 与 "$*" 的区别

```bash
#!/bin/bash
echo 'echo "$@" 输出:'; for arg in "$@"; do echo "[$arg]"; done
echo 'echo "$*" 输出:'; for arg in "$*"; do echo "[$arg]"; done
```

```bash
$ bash args.sh one two "hello world"
echo "$@" 输出:
[one]
[two]
[hello world]

echo "$*" 输出:
[one two hello world]
```

> **结论**：几乎所有情况下都应使用 `"$@"`，它能正确保留含空格和特殊字符的参数。

### 9.5 set 命令

`set` 用于操控 shell 选项和位置参数：

```bash
# 设置默认值，防止参数缺失时报错
set -- "默认用户" "默认密码"
echo "用户: $1, 密码: $2"

$ bash script.sh          # 使用默认值
用户: 默认用户, 密码: 默认密码

$ bash script.sh alice 123  # 使用传入值
用户: alice, 密码: 123

# 开启选项
set -e   # 命令失败时立即退出脚本
set -u   # 使用未定义变量时报错
set -x   # 打印每条命令及其参数（调试用）
set -o pipefail  # 管道中任意命令失败即退出
```

### 9.6 getopts 命令行选项解析

#### 9.6.1 为什么需要 getopts

先看一个常见场景：你在终端执行一个脚本，传入这样的参数：

```bash
./deploy.sh -v -o /tmp/build myapp.tar.gz
```

参数分两类：
- **选项**（flag）：`-v`（verbose，详细输出）、`-o`（指定输出目录）
- **普通参数**：文件名 `myapp.tar.gz`

用位置参数 `$1` `$2` 依次访问的话，必须记住顺序：`-v` 必须在前、`-o` 必须接路径。如果用户写成 `./deploy.sh myapp.tar.gz -v -o /tmp/build`， `$1` 就变成了文件名，脚本就会出错。

`getopts` 就是用来解决这个问题的：它专门解析命令行选项，支持任意顺序，还能识别哪个选项带了参数。

#### 9.6.2 语法基础

```bash
while getopts "vo:" opt; do
    case "$opt" in
        v)  echo "verbose 模式" ;;
        o)  echo "输出目录: $OPTARG" ;;
    esac
done
shift $((OPTIND - 1))
echo "剩余参数: $@"
```

**三个核心要素**：

| 要素 | 说明 |
|------|------|
| `optstring` | 选项定义字符串，如 `"vo:"` 表示 -v 和 -o，**-o 后面的冒号表示需要参数** |
| `varname` | 每轮循环中，当前匹配到的选项字母存入此变量（如 `opt`） |
| `$OPTARG` | 当选项需要参数时，参数值存在这个变量里 |
| `$OPTIND` | 下一个待处理参数的位置（处理完后用它配合 shift） |

**optstring 规则**：

| optstring 示例 | 含义 |
|---------------|------|
| `"v"` | 只有 `-v` 一个选项，不带参数 |
| `"vo:"` | `-v` 不带参数，`-o` 必须带参数（冒号表示"需要值"） |
| `":v"` | 第一个冒号表示**忽略错误提示**，`-v` 不带参数 |
| `"v:o:"` | `-v` 和 `-o` 都必须带参数 |

> **记忆技巧**：冒号 `:` 在 optstring 中跟在哪个字母后面，哪个选项就需要参数。

#### 9.6.3 工作原理图解

以 `./script.sh -v -o /tmp a.txt b.txt` 为例：

```
getopts "vo:" opt  逐步执行过程：

┌─────────────────────────────────────────────────────────────┐
│  optstring = "vo:"                                          │
│             ↑  ↑                                            │
│             │  └── o 后有冒号 → -o 需要参数                  │
│             └────── v 后无冒号 → -v 不需要参数               │
└─────────────────────────────────────────────────────────────┘

第一轮循环：
  $OPTIND=1，当前参数=-v
  getopts 发现 -v 匹配 → opt="v"，返回真，进入 case 处理 verbose 模式

第二轮循环：
  $OPTIND=1（还没动），当前参数=-o
  getopts 发现 -o 匹配，且需要参数
  → opt="o"，$OPTARG="/tmp"，$OPTIND 移动到下一个位置

第三轮循环：
  $OPTIND=3，当前参数=a.txt
  getopts 发现不是以 - 开头 → 返回假，退出循环
  此时 $OPTIND=3

shift $((OPTIND - 1))  → shift 2 → 参数列表变为 ["a.txt", "b.txt"]
$@ 现在 = "a.txt b.txt"（剩下的都是非选项参数）
```

#### 9.6.4 完整注释示例

```bash
#!/bin/bash
# deploy.sh — 部署脚本，演示 getopts 用法
# 用法：./deploy.sh [-v] -o <输出目录> <文件...>

# 默认值
VERBOSE=false
OUTPUT_DIR="/tmp"

while getopts "vo:" opt; do
    case "$opt" in
        v)
            VERBOSE=true
            ;;
        o)
            OUTPUT_DIR="$OPTARG"       # $OPTARG 拿到 -o 后面的参数
            ;;
        \?)                            # 匹配到未知选项，或缺少必需的参数
            echo "未知选项: -$OPTARG" >&2
            exit 1
            ;;
    esac
done

shift $((OPTIND - 1))                  # 跳过已解析的选项，剩下是文件名

# 此时 "$@" 就是 ./deploy.sh 后面不再是选项的参数列表
if [ "$#" -eq 0 ]; then
    echo "错误：请指定至少一个文件"
    exit 1
fi

echo "输出目录: $OUTPUT_DIR"
$VERBOSE && echo "Verbose 模式已开启"

for file in "$@"; do
    echo "部署文件: $file"
done
```

```bash
$ bash deploy.sh -v -o /var/www file1.txt file2.txt
输出目录: /var/www
Verbose 模式已开启
部署文件: file1.txt
部署文件: file2.txt

$ bash deploy.sh file1.txt -o /backup -v   # 选项顺序无所谓
输出目录: /backup
Verbose 模式已开启
部署文件: file1.txt
```

#### 9.6.5 与 shift $((OPTIND - 1)) 的配合

`shift $((OPTIND - 1))` 是固定的结尾模式，必须放在 `while getopts` 循环之后：

```
OPTIND = 下一个待处理的参数位置
shift $((OPTIND - 1)) = 把指针跳过所有已解析的选项
结果：$@ 现在只包含非选项的普通参数（如文件名）
```

> **为什么不用 shift 直接在循环里做？**
> 因为 `getopts` 自动维护 `OPTIND`，不需要也不应该手动 shift。用 `shift $((OPTIND - 1))` 一次性跳过所有选项才是正确做法。

#### 9.6.6 常见错误

```bash
# 错误一：忘记写 \?) 分支（未知选项时不会报错）
while getopts "vo:" opt; do
    case "$opt" in
        v) ;;
        o) ;;
    esac
done

# 错误二：optstring 中选项带参数但没写冒号
while getopts "vo" opt; do    # o 需要参数，但忘了写冒号
    case "$opt" in
        o) echo "$OPTARG" ;;  # $OPTARG 会是空的！
    esac
done

# 错误三：忘记写最后的 shift $((OPTIND - 1))
# 结果：$@ 还包含选项如 "-v"，当作文件名处理，会出错
```

## 10. 函数

### 10.1 定义与调用

Bash 函数有两种定义方式，效果完全相同：

```bash
# 方式一：function 关键字（可移植性稍差，但语义更明确）
function greet {
    echo "你好！"
}

# 方式二：圆括号语法（POSIX 标准，更通用）
greet() {
    echo "你好！"
}
```

```bash
# 调用函数：直接写函数名（不加括号或 $）
greet              # 输出：你好！
```

### 10.2 参数传递与返回值

**参数传递**：函数内部通过 `$1`, `$2`... 获取参数，与脚本参数独立：

```bash
greet_user() {
    echo "你好，$1！今天感觉如何？"
}

greet_user "Alice"   # 输出：你好，Alice！今天感觉如何？
greet_user "Bob"     # 输出：你好，Bob！今天感觉如何？
```

**返回值**：函数通过 `return` 返回退出状态码（0~255），通过 `echo` 返回数据：

```bash
is_even() {
    num=$1
    if [ $((num % 2)) -eq 0 ]; then
        return 0   # 偶数，返回 0 表示成功
    else
        return 1   # 奇数，返回非 0
    fi
}

# 调用并检查返回值
is_even 4
if [ $? -eq 0 ]; then
    echo "4 是偶数"
fi

# 返回字符串数据（通过命令替换捕获）
get_username() {
    echo "alice"
}

name=$(get_username)
echo "用户名: $name"
```

### 10.3 局部变量与全局变量

默认情况下，函数内创建的变量是全局的，会影响函数外部：

```bash
name="全局变量"

demo() {
    # 未声明 local，默认使用全局变量
    name="被函数修改了"
    echo "函数内部: $name"
}

demo
echo "函数外部: $name"
# 函数外部的 name 也被修改了！
```

使用 `local` 关键字限制变量作用域：

```bash
name="Alice"

greet() {
    local name="Bob"    # 仅在函数内有效
    echo "函数内: $name"  # 输出：Bob
}

greet
echo "函数外: $name"      # 输出：Alice（不受影响）
```

**最佳实践**：函数内变量一律加 `local`，避免意外修改全局状态。

### 10.4 函数库与 source

将常用函数保存到单独文件，通过 `source`（或 `.`）加载：

```bash
# 文件：lib/math.sh
add() {
    local a=$1 b=$2
    echo $((a + b))
}

multiply() {
    local a=$1 b=$2
    echo $((a * b))
}
```

```bash
# 文件：main.sh
#!/bin/bash
source lib/math.sh

result=$(add 10 20)
echo "10 + 20 = $result"

result=$(multiply 5 6)
echo "5 × 6 = $result"
```

### 10.5 递归函数

Bash 函数支持递归调用：

```bash
# 计算阶乘
factorial() {
    local n=$1
    if [ "$n" -le 1 ]; then
        echo 1
    else
        local prev=$(factorial $((n - 1)))
        echo $((n * prev))
    fi
}

echo $(factorial 5)   # 输出：120
```

## 综合练习

编写一个日志处理脚本，综合运用本阶段知识点：

```bash
#!/bin/bash
# log_processor.sh — 日志处理工具

# 加载函数库
source libs/color.sh  # 假设包含带颜色的输出函数

# 默认配置
VERBOSE=false
OUTPUT_DIR="/tmp/logs"

# 显示用法
usage() {
    echo "用法: $0 [-v] [-o 目录] 文件..."
    echo "  -v    详细输出"
    echo "  -o    指定输出目录"
}

# 解析参数
while getopts "vo:" opt; do
    case "$opt" in
        v) VERBOSE=true ;;
        o) OUTPUT_DIR="$OPTARG" ;;
        *) usage; exit 1 ;;
    esac
done
shift $((OPTIND - 1))

# 检查参数
if [ "$#" -eq 0 ]; then
    echo "错误：必须指定至少一个日志文件"
    usage
    exit 1
fi

# 确保输出目录存在
[ -d "$OUTPUT_DIR" ] || mkdir -p "$OUTPUT_DIR"

# 统计函数
count_errors() {
    local file=$1
    local count=$(grep -c "ERROR" "$file" 2>/dev/null || echo 0)
    echo "$count"
}

# 处理每个文件
total_errors=0
for logfile in "$@"; do
    if [ ! -f "$logfile" ]; then
        echo "警告：$logfile 不存在，跳过"
        continue
    fi

    errors=$(count_errors "$logfile")
    total_errors=$((total_errors + errors))

    filename=$(basename "$logfile")
    output_file="$OUTPUT_DIR/${filename}.report"

    {
        echo "=== 日志分析报告 ==="
        echo "文件: $logfile"
        echo "总行数: $(wc -l < "$logfile")"
        echo "错误数: $errors"
        echo "警告数: $(grep -c "WARN" "$logfile" 2>/dev/null || echo 0)"
    } > "$output_file"

    $VERBOSE && echo "已处理: $logfile → $output_file"
done

echo "处理完成。共发现 $total_errors 个错误。"
```

**用法示例：**

```bash
$ bash log_processor.sh -v -o /tmp/reports app.log database.log
已处理: app.log → /tmp/reports/app.log.report
已处理: database.log → /tmp/reports/database.log.report
处理完成。共发现 5 个错误。
```
